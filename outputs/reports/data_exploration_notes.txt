5. UNIT MISMATCHES & WHAT IS / IS NOT COMPARABLE
==============================================================================
  Effective Rent levels (Table 1):
    file 1 = $/unit (e.g. $1,174)
    file 3 = $/SQFT  (e.g. $1.33)
    NOT COMPARABLE. Use Table 2 (indexed % change) for cross-vintage analysis.

  Effective Rent Growth (file 2):
    Quarterly growth rate fraction (~0.005 per quarter typical). Comparable
    to itself; not directly to the indexed levels in files 1/3 without
    transformation.

  Asking Rent levels:
    file 1 Table 1 = $/unit
    file 3 Table 1 = $/unit (also unit-level, NOT $/sqft) — checked: Akron
       $1,240/Q2 2026 vs $1,181/Q2 2025; both unit-level.
    file 2 "Market Asking Rent" = $/SQFT (Akron $0.88 etc.)
    Tables 2 in files 1 & 3 are comparable indexed % change.

  Sale Price:
    file 1 Forward Sale Price Table 1 = an INDEX (e.g. Akron 222.26)
    file 3 Forward Sale Price Table 1 = INDEX same scale
    file 2 Market Sale Price Index = INDEX same scale
    These appear directly comparable.

  Employment & Supply (Inventory) & Population:
    All files report LEVELS in equivalent units (jobs, units, persons).

  Income:
    file 1 / file 3 Income Growth Table 1 = $/year per household
    file 2 Median Household Income = $/year per household
    Comparable.

  UPP:
    file 1 / file 3 derive UPP = inventory_units / population
    file 2 has UPP directly. Numerically aligned (~0.04-0.05).
