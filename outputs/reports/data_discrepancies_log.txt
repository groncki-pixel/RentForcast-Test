DATA DISCREPANCIES LOG
======================
Running log of data issues, assumptions, unit mismatches, and naming
inconsistencies encountered during the analysis pipeline. Append to this
file as new issues are discovered.

----- Step 0: Exploration -----

[1] Effective Rent-Forward Table 1 unit mismatch (already known per task).
    file 1: $/unit (e.g. $1,174). file 3: $/sqft (e.g. $1.33).
    Action: only Table 2 (indexed % change) used for cross-file comparison.

[2] Market name suffixes vary across sheets within a single file:
    "Akron - OH" vs "Akron - OH USA" vs "Akron - OH (USA)".
    Action: applied a normalize_market() that strips trailing " USA" or
    " (USA)" / "(USA)" and trims/collapses whitespace. Canonical: "Akron - OH".

[3] File 2 README claims "Poughkeepsie - NY in FML but not in MSA Export".
    Verification: Poughkeepsie - NY IS present in every file 2 data sheet.
    All 150 markets present in all 3 files. README is inaccurate / stale.
    Action: ignore README claim; proceed with all 150 markets.

[4] File 2 README flags Income / Population / UPP as "MISSING DATA - needs
    manual fill". Verification: the cells contain numeric values with high
    coverage (Income 100% non-null, Population 100% non-null, UPP ~98%).
    Most likely "yellow = needs fill" referred to highlighting some cells
    in the source workbook, not to systemic emptiness. Action: use the data
    as-is and flag any actually-null cells per script.

[5] File 1 forward sheets have INCONSISTENT leading columns:
    - "Effective Rent-Forward" / "Population Growth Forward Table1" /
      "Forward Employment Table1" have extra cols [Property Class Name,
      Slice, As Of, Geography Name, Concept Name] before the quarter cols.
    - Other forward sheets (Asking Rent, Supply Forward, Income Growth,
      Forward Sale Price, Forward UPP) have just Geography Name in col 0.
    File 3 strips these extra cols entirely (Geography Name in col 0 always).
    Action: extractor detects the Geography column position dynamically.

[6] Stacked tables present in nearly every "Forward" / "Historical" sheet of
    files 1 and 3:
    - Table 1 (rows 1-150): raw level values
    - Table 2 (rows 154-303): indexed % change (anchored to 1.00 at est qtr)
    - Table 3 (rows 306+): aggregate 1/3/5/10-year growth rates / ranks
    Some sheets (UPP Historical) have 5+ stacked sub-blocks.
    Action: split by the blank row at row 151 and the repeated header at
    row 153; label the splits "_levels", "_pct_change", "_aggregate".

[7] Asking Rent has weight 0.00 in row 1 of Summary Sheet (verified col index
    is empty in the weights row for asking rent block). Asking rent is
    tracked / ranked / displayed but contributes 0% to the weighted average.
    Action: when re-running weighted averages, treat asking-rent weight = 0.

[8] The "Effective Rent Historical" sheet in file 1 is a 152x8 mini-table of
    just 5/10-year growth rankings. In file 3 it is a full 607x42 quarterly
    time series 2016-2036. Different schemas under the same sheet name.
    Action: handle each separately; do not assume file 1 and file 3 historical
    sheets share a layout.

[9] Estimate quarters differ:
    - file 1 forward sheets baseline at "2025 Q2 EST"
    - file 3 forward sheets baseline at "2026 Q2 EST"
    Cross-file Table 2 comparisons should align by lead/lag quarter offset
    rather than calendar quarter when needed.

[10] The "As Of" tag inside Effective Rent-Forward Table 2 says "2024 Q3"
     in the original and a different quarter in file 3, but values are
     anchored to the estimate quarter (= 1.00 at 2025 Q2 / 2026 Q2). The
     "As Of" appears to be cosmetic. Action: treat Table 2 values as
     rebased to the estimate quarter regardless of "As Of" label.

[11] Construction Starts (file 2) is 67% non-null — many small markets have
     long zero stretches. Action: in script 6, treat zeros as actual zero
     (= no construction that quarter), not missing.

[12] File 3 Population Growth Forward has only 21 columns vs 46/40 in other
     forward sheets. Likely truncated or only 5 yr of forwards rather than
     10. Action: extract whatever quarters are present; do not pad missing.

[13] File 3 Summary Sheet has a duplicated "2025 Analysis" block in cols
     49-52 that mirrors the file 1 final ranking. We use this only as a
     cross-check; primary source for orig rankings remains file 1.
